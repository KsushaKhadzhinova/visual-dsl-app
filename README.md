DiagramCode — Visual DSL DiagrammerПрофессиональная веб-ориентированная среда разработки (IDE) для визуального моделирования диаграмм с использованием предметно-ориентированных языков (DSL). Проект выполнен в рамках учебного цикла лабораторных работ по дисциплине "Проектирование веб-интерфейсов".Исполнитель: Ксения ХаджиноваВариант: 13🚀 Статус выполнения лабораторных работЛабораторная работа №5: Разработка интерактивного интерфейса на JavaScriptМодульная архитектура (ES6+): Код разбит на независимые модули (utils/helpers.js, components/modal.js, components/editor.js, script.js), которые импортируются друг в друга.Работа с DOM: - Динамическая генерация номеров строк в редакторе при вводе текста.Динамическое создание и удаление всплывающих уведомлений (Toast) с использованием document.createElement, appendChild и remove().Добавление сообщений об ошибках при валидации форм напрямую в DOM дерево.Обработка событий и Делегирование: - Использован паттерн делегирования событий: на объект document (или body) навешен единый слушатель кликов, который управляет открытием и закрытием всех модальных окон через атрибут aria-controls.Обработаны события: click, input, keydown, scroll, change, blur, submit.LocalStorage: Реализовано автосохранение кода из редактора (diagramcode_draft). При перезагрузке страницы код восстанавливается автоматически.Валидация форм: Строгая валидация формы "AI Prompt". Поле email проверяется регулярным выражением (Regex) при событии blur (потеря фокуса), а вся форма проверяется при submit.Асинхронность: Кнопка "Run" имитирует отправку данных на сервер с помощью объекта Promise, конструкции async / await и блока try / catch / finally, блокируя кнопку и накладывая визуальный фильтр размытия на диаграмму во время ожидания.Лабораторная работа №4: Адаптивность и Методология БЭМАрхитектура CSS: Код организован строго по методологии БЭМ (Блок-Элемент-Модификатор). Стили разделены на модули.Современные сетки: Использован CSS Grid Layout для двухколоночной структуры рабочей области и Flexbox для панелей инструментов.Адаптивность (Mobile-First): Реализовано мобильное меню в виде боковой панели (Drawer) и система вкладок (Tabs) для переключения между кодом и диаграммой на смартфонах.Лабораторная работа №3: Семантика и ДоступностьСемантическая верстка: Использованы нативные теги HTML5 (<header>, <main>, <section>, <aside>, <footer>, <figure>).Доступность (Accessibility): Внедрен Skip-link для быстрого перехода к редактору при навигации с клавиатуры. Применены ARIA-атрибуты.Микроразметка: Интегрированы словари Schema.org (SoftwareApplication, Person, Organization).Лабораторная работа №2: Окружение и инфраструктураИнициализирован проект через npm init.Настроены скрипты для запуска сервера разработки (live-server).Настроен .gitignore для исключения системных файлов и папки node_modules.🛠 Технологический стекMarkup: HTML5 (Semantic & valid, WAI-ARIA)Styling: CSS3 (Variables, Grid, Flexbox, BEM, Animations)Scripting: Vanilla JavaScript (ES6 Modules, LocalStorage, Async/Await)Icons: Lucide Icons / SVGDev Tools: npm, live-server📂 Структура проектаVisualDSL_Project/
